## 二、日常操作

### 1）启动 / 重启服务
双击 **`start-meeting.bat`**（或右键 `start-meeting.ps1` → 用 PowerShell 运行）。
脚本会自动：停旧进程 → 设好数据目录 → 后台重启 → 健康检查。

### 2）确认服务在线
浏览器打开 `http://localhost:8080/` 能看到会议首页即正常。
或看日志末尾是否有 `Meeting server running: http://localhost:8080`。

### 3）改了代码后怎样生效
- **前端**（`public/index.html`）：静态文件，刷新浏览器即生效，**无需重启**。
- **后端**（`server.js`）：改完必须重启（跑一次 `start-meeting.bat`）。

## 三、⚠️ 长久稳定的唯一软肋：临时隧道

现在对外用的是 **Cloudflare 免费临时隧道**（`finishing-feelings-null-replace.trycloudflare.com`）。
**问题**：这个域名是随机的，**隧道进程或机器一重启，域名就变**，旧链接全部失效。

### 要做到"长久稳定、链接永不变"，三选一：

**方案 A（推荐·免费·固定域名）：Cloudflare 命名隧道**
1. 有一个自己的域名，托管到 Cloudflare。
2. 装 `cloudflared`，`cloudflared tunnel login` → `cloudflared tunnel create meeting`。
3. 配 `config.yml` 把 `meeting.你的域名.com` 指向 `http://localhost:8080`。
4. `cloudflared tunnel run meeting`（可注册成 Windows 服务开机自启）。
→ 域名固定、永不变、免费、带 HTTPS。**这是最省钱又最稳的选择。**

**方案 B（最稳·要钱）：上 VPS**
按压缩包内 `README.md` 的 docker-compose 部署到云服务器（含自托管 coturn TURN，手机 4G/5G 也能连）。域名固定、7×24 在线、不依赖你本机开机。约 ¥60–100/年 VPS。

**方案 C（临时凑合）：继续用免费临时隧道**
每次机器重启后，隧道域名会变，需要重新生成链接发群里。适合短期测试，不适合长期。

> 需要我帮你上**方案 A（固定域名隧道）**随时说一声——这是"长久使用"性价比最高的一步。

## 四、开机自启（可选，让服务随 Windows 启动）

把 `start-meeting.bat` 的快捷方式放进启动文件夹：
1. `Win + R` → 输入 `shell:startup` → 回车。
2. 把 `start-meeting.bat` 的快捷方式拖进去。
→ 以后开机自动拉起会议服务（隧道另配，见方案 A）。

## 六、常见问题

| 症状 | 排查 |
|---|---|
| 打不开会议 | 先看 `http://localhost:8080/` 通不通；不通就跑 `start-meeting.bat` |
| 登录/注册报错 | 后端没重启（跑 `start-meeting.bat`）；或数据目录不可写（看 err 日志有无 `EPERM`） |
| 链接失效 | 多半是隧道域名变了（临时隧道通病）→ 上方案 A 固定域名 |
| 远端黑屏 | 对方没开摄像头会显示占位提示，属正常；真黑屏查对方网络/TURN |
| 满 100 人进不去 | 达到容量上限，属设计行为 |
